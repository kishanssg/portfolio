# Portfolio Design & Development Rules

## Core Philosophy
This portfolio must reflect premium craftsmanship, thoughtful design decisions, and authentic personality. Every element should feel intentional, polished, and distinctly human-crafted.

## ðŸŽ¨ Design Principles

### Visual Identity
- **Color Palette**: Use a sophisticated, limited color palette (3-5 colors maximum)
  - Choose colors that reflect your personality and brand
  - Ensure WCAG AA accessibility standards (4.5:1 contrast ratio minimum)
  - Avoid overused gradients (purple-to-blue, etc.)
  - Consider using a signature accent color unique to your brand

### Typography
- **Font Pairing**: Maximum 2-3 typefaces across the entire site
  - One for headings (display/serif for character)
  - One for body text (highly readable sans-serif)
  - Optional: One for accents/code
- **Hierarchy**: Clear visual hierarchy with intentional sizing (not just h1, h2, h3 defaults)
- **Line Height**: 1.5-1.7 for body text, 1.2-1.4 for headings
- **Font Size**: Minimum 16px for body text, scale up proportionally
- **Custom Fonts**: Use quality fonts from Google Fonts, Adobe Fonts, or self-hosted
- **Avoid**: Generic system fonts unless part of a deliberate design system

### Spacing & Layout
- **Consistent Spacing Scale**: Use a modular scale (e.g., 4px, 8px, 16px, 24px, 32px, 48px, 64px)
- **Breathing Room**: Generous whitespace - don't cram content
- **Grid System**: Use a consistent grid (12-column or custom)
- **Asymmetry**: Strategic use of asymmetrical layouts for visual interest
- **Max Width**: Contain content (1200-1400px max-width for readability)

### Animations & Interactions
- **Purposeful Motion**:  Animations should enhance UX, not distract
- **Timing Functions**: Use easing curves (ease-in-out, cubic-bezier) - avoid linear
- **Duration**: 200-400ms for most interactions, 600-1000ms for page transitions
- **Reduce Motion**: Always respect `prefers-reduced-motion`
- **Hover States**: Every interactive element must have clear hover/focus states
- **Micro-interactions**: Subtle feedback for clicks, hovers, scrolls
- **Page Transitions**: Smooth, meaningful transitions between pages/sections

### Visual Elements
- **Icons**: Use a consistent icon set (Heroicons, Lucide, Phosphor, or custom)
- **Images**: High-quality, optimized, and purposeful
  - Use WebP format with fallbacks
  - Lazy loading for performance
  - Proper aspect ratios
- **Graphics**: Custom illustrations/graphics over stock imagery when possible
- **Shadows**: Subtle, realistic shadows - avoid harsh drop-shadows
- **Borders**: Use sparingly and with purpose
- **Glassmorphism/Neumorphism**: Use tastefully if at all

## ðŸ’» Development Standards

### Code Quality
- **Clean Code**:  Readable, well-commented, and semantic
- **Component Structure**: Small, reusable, single-responsibility components
- **Naming Conventions**: Clear, descriptive variable/function names
- **No Dead Code**: Remove commented-out code and unused imports
- **Consistent Style**: Use Prettier and ESLint with strict rules
- **Type Safety**: Use TypeScript with strict mode enabled

### Performance
- **Core Web Vitals**: Optimize for LCP, FID, CLS
  - LCP: < 2.5s
  - FID: < 100ms
  - CLS: < 0.1
- **Bundle Size**: Keep JavaScript bundles under 200KB (initial)
- **Image Optimization**: Compress images, use next-gen formats
- **Code Splitting**: Lazy load routes and heavy components
- **Caching**: Implement proper caching strategies
- **Fonts**:  Preload critical fonts, use font-display: swap

### Accessibility (WCAG 2.1 AA Minimum)
- **Semantic HTML**:  Proper use of headings, landmarks, lists
- **Keyboard Navigation**: Full keyboard accessibility
- **Screen Readers**:  ARIA labels where necessary, alt text for images
- **Focus Indicators**: Clear, visible focus states
- **Color**:  Don't rely solely on color to convey information
- **Forms**: Proper labels, error messages, validation

### SEO & Meta
- **Meta Tags**: Proper title, description, OG tags
- **Structured Data**: JSON-LD schema for personal/professional info
- **Sitemap**: Generate and submit sitemap. xml
- **Robots.txt**: Configure appropriately
- **URLs**: Clean, descriptive URLs
- **Performance**: Fast load times improve rankings

### Responsive Design
- **Mobile-First**: Design and develop mobile-first
- **Breakpoints**: Common breakpoints (sm:  640px, md: 768px, lg: 1024px, xl: 1280px)
- **Touch Targets**: Minimum 44x44px for touch elements
- **Testing**: Test on real devices, not just browser dev tools
- **Orientation**: Consider landscape and portrait modes

## ðŸ“± Content Guidelines

### Writing Style
- **Authentic Voice**: Write in your own voice, not corporate speak
- **Concise**:  Clear and to the point - no fluff
- **Active Voice**: Use active voice over passive
- **Show, Don't Tell**: Demonstrate skills through projects, not just lists
- **Proofreading**: Zero typos, proper grammar

### Project Showcases
- **Quality Over Quantity**: 4-6 excellent projects > 20 mediocre ones
- **Case Studies**: Tell the story - problem, solution, impact, tech stack
- **Visuals**: High-quality screenshots, demos, or videos
- **Live Demos**: Link to live versions when possible
- **Code**:  Link to GitHub repos (for appropriate projects)
- **Impact**: Quantify results when possible (metrics, testimonials)

### About/Bio Section
- **Personal**:  Show personality and authenticity
- **Professional**: Balance casual with professional
- **Story**: Share your journey, what drives you
- **Skills**: Show don't just list - integrate into narrative
- **Photo**: Professional but approachable photo

### Contact/CTA
- **Clear CTA**: Make it obvious how to reach you
- **Multiple Channels**: Email, LinkedIn, GitHub, etc.
- **Response Time**: Set expectations (if you mention it)
- **Professional Email**: Use custom domain if possible

## ðŸ›  Technical Stack Recommendations

### Suggested Technologies
- **Framework**: Next.js, Astro, SvelteKit, or Nuxt
- **Styling**: Tailwind CSS, CSS Modules, or Styled Components
- **Animations**: Framer Motion, GSAP, or CSS animations
- **Forms**: React Hook Form, Formik, or native HTML5
- **Analytics**: Plausible, Fathom, or Google Analytics
- **Hosting**: Vercel, Netlify, or Cloudflare Pages
- **CMS**: Optional - Sanity, Contentful, or MDX for blog

### Dependencies
- Keep dependencies minimal and purposeful
- Regularly update dependencies for security
- Audit bundle size impact before adding new packages
- Prefer native solutions over heavy libraries when possible
